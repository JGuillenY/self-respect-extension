## Build

```bash
# Install dependencies
pnpm install

# Build everything (TypeScript compile + locale minification + ZIP packaging)
pnpm run build
# or directly:
bash ./build.sh
```

The build script (`build.sh`):
1. Wipes `dist/` and recreates it
2. Copies static assets (icons, HTML, manifest)
3. Minifies `src/_locales/*/messages.json` → `dist/_locales/` via `minify-locales.js`
4. Bundles and minifies the three entry points with esbuild (targeting Chrome 58+)
5. Packages `dist/` into `dist.zip`

There is no watch mode or test suite. To test, load `dist/` as an unpacked extension in Chrome (`chrome://extensions/` → Developer mode → Load unpacked).

## Architecture

This is a Manifest V3 Chrome extension with three entry points bundled independently by esbuild:

### Entry points
- **`src/content.ts`** — Content script injected into every page. On load, checks `window.location.hostname` against predefined and custom blocked domains, then renders a full-page overlay matching the configured blocking level.
- **`src/popup.js`** — Toolbar popup. Reads stats and settings from `chrome.storage.sync` and renders the quick-view UI.
- **`src/settings.js`** — Full settings page opened in a new tab. Manages all user preferences and custom domain list.

### Core modules (TypeScript, imported by `content.ts`)
- **`src/constants.ts`** — Static domain blocklist organized into three `DomainCategory` objects (Adult Content, Social Media, Gambling), each with a `redirectTo` URL. Exports `isDomainBlocked()` and `getRedirectUrlForDomain()` for synchronous lookups.
- **`src/storage.ts`** — All `chrome.storage.sync` reads/writes. Defines `UserSettings` and `BlockedSite` types. The `isDomainBlocked()` here checks only custom domains (enabled flag + normalization); predefined category checks happen in `constants.ts`.
- **`src/puzzle.ts`** — Unused stub (sudoku logic lives inline in `content.ts`).
- **`src/types.ts`** — Empty (types are co-located with their modules).

### Blocking levels (implemented in `content.ts`)
| Level | Behavior |
|-------|----------|
| `soft` | Overlay with countdown timer, redirect and cancel buttons |
| `puzzle` | Full-page Sudoku puzzle; site accessible only after solving |
| `hard` | Overlay with no bypass; only "Go to Healthier Alternative" |

The Sudoku generator runs entirely in-browser: generates a complete valid board, removes 55 cells (hard difficulty), and validates against the stored solution on each number entry.

### Localization
All user-visible strings in `content.ts`'s overlay HTML (except the Sudoku puzzle block, which uses hardcoded English) must use `chrome.i18n.getMessage("keyName")`. Source strings live in `src/_locales/<locale>/messages.json`; `dist/_locales/` contains minified copies generated by `minify-locales.js`. Supported locales: `en`, `es`, `pt_BR`, `ru`, `zh_CN`.

## Key constraints

- `popup.js` and `settings.js` are plain JavaScript (not TypeScript) — excluded from `tsconfig.json`.
- `chrome` is declared `any` in TypeScript files to avoid requiring `@types/chrome` at compile time.
- esbuild, not `tsc`, performs the actual compile+bundle. `tsconfig.json` is used for type-checking only.
- Adding a new locale requires creating `src/_locales/<locale>/messages.json` and adding the corresponding `dist/_locales/<locale>/` directory in `build.sh`.
